Functions


Erlava relies heavily on two core concepts: functions and patterns. Understanding these concepts is essential for writing effective Erlava programs.


Functions are reusable blocks of code that perform specific tasks. They take arguments (inputs) and return a value (output) or perform actions without returning a value. Here's are some common functions

      
        add(X,Y) -> X + Y. %% Adds X and Y. Variables next chapter :)

Erlava support multiple dispatch of functions using patterns. Patterns are used in function definitions and in conditional statements like case expressions. They define the expected format of the data passed to the function or being matched against. Here are some examples: 

      
        fac(0) -> 1. %% Factorial called with 0 return 1.
        fac(1) -> 1. %% Factorial called with 1 return 1.
        fac(N) -> N * fact(N - 1). %% Recursively call factorial.

        greet(president) -> io:fwrite("hello, president").
        greet(You) when string:length(You) > 0 -> io:fwriteln("Hello %s, Howzit", You).
        
        head(First|Others) -> First.
